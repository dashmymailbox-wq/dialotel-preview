Le problème : container lavande dans WordPress

Le fond dégradé du plugin s'affiche comme une boîte car .vt-app est piégé dans le contenu WP

Preview HTML — Correct ✓
vt-app = plein viewport
fond lavande invisible car full-width
Plugin WordPress — Problème ✗
vt-app = contraint
fond lavande visible comme container

← fond blanc WP visible tout autour →

3 approches pour corriger

A

Breakout CSS

Donner à .vt-app une position qui le sort du flux WP (negative margins ou width:100vw). Simple, mais peut casser certains thèmes.

B

Fond transparent + overlay body

Supprimer le fond lavande de .vt-app et injecter le dégradé directement sur le body via le shortcode. Propre, sans breakout.

C

Renforcer les resets shortcode

Ajouter les sélecteurs manquants du thème Dialotel dans $hide_css pour forcer tous les ancêtres à 100% width et 0 padding. Plus ciblé.
